# 00 可迁移项目设计

## 目标

让一个新手在另一台电脑上也能复现完整综述流程。

## 分层设计

### 第 1 层：必须人工确认

- 综述主题和边界；
- 纳入/排除标准；
- 文献是否真正相关；
- 结论是否被证据支持；
- Word 中 Zotero 引文是否刷新正常。

### 第 2 层：可以半自动

- 生成检索式；
- PubMed/OpenAlex 初检索；
- RIS/CSV 导出；
- evidence matrix 初稿；
- 综述表格草稿；
- 章节草稿；
- Word 字段审计。

### 第 3 层：可以一键生成

- 项目目录；
- 模板文件；
- 提示词；
- 质量检查清单；
- 检索结果 CSV/RIS；
- Markdown 初稿骨架。

## 为什么不能完全无人值守？

1. Zotero Connector 和 Word 插件需要本机 UI 权限；
2. 数据库访问权限、PDF 下载权限因学校/医院网络而异；
3. 高质量综述必须人工判断证据和争议；
4. Zotero 字段必须经过 Word/Zotero Refresh 验证。

因此本项目采用“**一键初始化 + 分阶段自动化 + 强制质控**”模式。
